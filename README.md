# HP Plugin

WordPressサイト向けの独自プラグインを管理するリポジトリです。

既存テーマは極力変更せず、プラグインの有効化・無効化で機能を切り替えられる構成を基本とします。

## 管理対象

- 開催状況表示プラグイン
- 共通画像トリミング
- モバイル表示管理
- 今後追加するサイト改修用プラグイン

## ディレクトリ構成

```text
plugins/
├─ garden-opening-status/
├─ universal-image-cropper/
└─ mobile-layout-manager/
```

各プラグインは、WordPressの `wp-content/plugins/` にそのまま配置できるフォルダ構成で管理します。

## 開発方針

- 既存テーマファイルを直接編集しない
- プラグイン停止で元の表示へ戻せるようにする
- PC・スマホの表示を分けて管理できるようにする
- 管理画面プレビューと公開画面の表示を一致させる
- 修正は原則として1件ずつ行う
- 動作確認前に複数の機能を同時変更しない

## 更新方法

1. GitHub上のプラグインコードを更新
2. プラグインごとにZIPを作成
3. WordPress管理画面から同名ZIPをアップロード
4. 「現在のものをアップロードしたものに置き換える」を選択
5. 管理画面プレビューと公開画面を確認

## バージョン管理

プラグインのメインPHPにある `Version` を更新します。

```php
/**
 * Plugin Name: Example Plugin
 * Version: 1.0.0
 */
```

目安：

- 不具合修正: `1.0.0` → `1.0.1`
- 小機能追加: `1.0.0` → `1.1.0`
- 大規模変更: `1.0.0` → `2.0.0`

## ZIP作成時の注意

ZIPを展開した直下に、プラグインフォルダが1つだけ入る形にします。

```text
plugin-name.zip
└─ plugin-name/
   ├─ plugin-name.php
   ├─ assets/
   └─ readme.txt
```

## 動作確認

最低限、次を確認します。

- プラグインの有効化でエラーが出ない
- 管理画面が開く
- 設定を保存できる
- PC表示が崩れない
- スマホ表示が崩れない
- プレビューと公開画面が一致する
- プラグイン停止で元の表示へ戻る

## 注意

このリポジトリには、パスワード、APIキー、メールアドレスなどの機密情報を保存しないでください。
